RBAC — Azure AD App Roles

RoleJWT ClaimPermissionsEnforcement
Owner SmartNest.Owner Full CRUD — homes, rooms, devices, rules, alerts, users within their home. Access to ReplayEvents. APIM JWT policy + Function role guard + homeId claim match
Technician SmartNest.Technician Read/write devices & device state; read-only on home/rules — scoped to their homeId. Function role guard + homeId claim must match device's homeId
Guest SmartNest.Guest Read-only on device state within their assigned homeId. Function role guard + homeId claim match; 403 on any mutation

JWT tokens carry roles (App Role) and homeId (optional claim). APIM validates token signature & audience. Functions enforce both role value and homeId matching.

Event Sourcing — Audit Service

Append-only Event Store

Every domain event on every topic is consumed by the audit subscription and written as an immutable AuditEntry to Cosmos DB audit-log.

Atomic sequenceNumber: A Cosmos DB transactional batch atomically increments the SequenceCounter document and inserts the AuditEntry in the same partition — zero gaps, zero duplicates.

AuditEntry schema

{
  "eventId":       "<uuid>",
  "eventType":     "DeviceStateChanged",
  "aggregateId":   "<deviceId>",
  "aggregateType": "Device",
  "occurredAt":    "2024-01-15T00:00:00Z",
  "actorId":       "<userId>",
  "homeId":        "<homeId>",
  "correlationId": "<uuid>",
  "sequenceNumber": 42,
  "payload":       { ... }
}

ReplayEvents HTTP endpoint (GET /audit/{aggregateId}?from={sequence}) is Owner-only — enforced at both APIM policy level and inside the Function (defence in depth).

Architecture Decision Records (ADRs)

ADR-001 — Functions over App ServiceConsumption plan within Free Tier (1M exec/month); native SB/Blob/Timer bindings eliminate boilerplate.
ADR-002 — Cosmos DB ServerlessPay-per-RU with no idle charge; flexible document model for heterogeneous aggregates. Provisioned 1000 RU/s ≈ $58/month.
ADR-003 — Service Bus over Event HubNamed subscriptions (Automation/Alert/Audit) allow independent consumption. Event Hub is telemetry-optimised, not pub/sub.
ADR-004 — Dedicated Audit ServiceCross-cutting append-only log avoids duplicating event store logic across 6+ services; enables future Cosmos DB change-feed projections.
ADR-005 — Entra ID over B2CApp Roles + optional claims (homeId) natively available free; B2C requires separate tenant + custom XML policies — heavy for a POC.
ADR-006 — Blob trigger sole media pathRemoves dual-path ambiguity between SB queue and Blob trigger. media-processing queue dropped. Idempotency via MediaMetadata doc check.
ADR-007 — Transactional batch for sequenceNumberAtomically increments SequenceCounter + inserts AuditEntry in same Cosmos DB partition. No gaps or duplicates vs etag retry loops.
ADR-008 — Cosmos DB Serverless capacityServerless billed only on consumed RUs — essentially free for low-volume dev/test. Provisioned throughput would burn trial credit unnecessarily.
